# Security Policy

## Supported Versions

The latest released version of `regsensitivity` receives security fixes.

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Use GitHub's private vulnerability reporting:

> [Report a vulnerability →](https://github.com/mikenguyen13/regsensitivity/security/advisories/new)

This opens a confidential channel visible only to repository
maintainers. Include:

* A short description of the vulnerability
* Steps to reproduce (a minimal example helps)
* The output of `sessionInfo()`

You can expect an acknowledgement within 7 days. Fixes are typically
released within 30 days as a patch version.

If GitHub's advisories feature is unavailable to you, you may instead
open a regular issue marked with the `security` label and a maintainer
will follow up privately.
